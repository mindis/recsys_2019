Conclusion¶

Overall, using demographics user information, it is possible to cover at most 3027 users with both age and region while leaving out 629 users that can be handled by non-personalized recommender or by using only information about the age (even a hybrid if possible). Then, leaving out 77 users that can be handled by non-personalized recommender.

Remember that these methods cannot be tested with validation set, because there is none. So, if you want it is only possible to test it via half-test set on Kaggle. This is just a gamble and you might overfit it when using information about demographics.

In conclusion, there are different mixture of ways to handle it:

  • Using ICM and try to find similar items for small user profiles
  • Using UCM (i.e. user demographics) to find a group of similar users (i.e. by clustering, ...) and provide similar recommendations for users with empty user profiles
  • Using non-personalized recommenders, i.e. Top-Popular (most easy way) for no user demographics

Non-personalized recommender can always be considered in some way, i.e. hybrid with UCM or ICM techniques
